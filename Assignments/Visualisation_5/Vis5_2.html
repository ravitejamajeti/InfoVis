Eyes Beat Memory : Animation VS Side by Side

Using Side by Side Views to compare both data

This charts presents the details about the HeadCount of students per major in years 2007 and 2008.

X-Axis - Major, Y-Axis - Head Count of Students, On the top left of Y-Axis - You can observe year for which you are checking statistics

By checking the above graphs now we are able to compare the statistics about the major's in both years easily. This is better than animation because there is no need to memorize any of the statistics to compare it with another

Got a comment for some other user of gist.github.com saying why not keep same years in same chart, so that we can compare from same chart without looking over different charts. I included my reply below explaining why i did not keep both the years in same chart.

"Yes, for small amounts of data like this we can implement it in same chart (Possibly using another chart instead of bar chart). But when there is enormous amount of data the chart may become clumsy and it might be difficult for user to find out specific data that he/she needs from chart. Again we can overcome this by including filters for chart, so that user can apply appropriate filters and keep only necessary data in chart. Also using this filters serves good only for some types of charts. For some type of charts which includes area to measure magnitude instead of position even if we apply filters, keeping 2 years in same chart will make chart clumsy and useless. So side by side charts will serve good in most of the cases. We can include a functionality from which user can generate only necessary graphs, keep them side by side and compare them. But my idea is to demonstrate the situation in which using animation is not a good idea."
